About ProjectLead

ProjectLead is an application that should help with organizing projects and teams. It offers easy to use UI and it should help administrator of a project/team to have a better understanding of what is going on with the project or some stage. There are two types of users: Admin and Regular User. Depending on what type of user is you account you will be able to see different things.

 General

On top of the application you will always see the topbar where you can find logo, with a home link, back button and submenu for updating your profile details or logging out.

 Dashboard

Dashboard is starting point of the application. Here you see a list of all projects and, if you are admin user, see a list of teams and latest comments. In order to begin with the app, you must create a project first. Then you can create teams which will be assigned to a project. After creating project/team you can click on the button and go further into the applicaiton. You can also remove project/team by clicking delete icon.

 Boards

Boards view allow you to create different boards for your selected project. You can view/edit/ delete any board here. It you choose view mode, then you are redirected to a Backlog.

 Backlog

Backlog serves as a place to gather all issues for a project board. Here you can see a full list of issues, even ones that are already in 'DONE' status. You can create new issues, edit or delete them. Once you are ready to start the Sprint, you can select which issues you want to assign, and then at the bottom of the issue list there will be a button for creating new Sprint. Here, just add Sprint name, dates and you are ready to go. In Backlog view you also get a submenu which contains following items: 'Backlog', 'Active Sprint', 'History', 'Reports'.

 Active Sprint

Once you successfully created Sprint, you and your users can start moving issues around through the defined board columns. You can check any issue, add comment to it, or delete it. Once sprint is done, you can choose to close it. If not it will auto closed when you create new Sprint.

 History

In this view you get to see all previous sprints, with dates, and sprint points and with the list of resolved issues per sprint.

 Reports

Here you will be able to see some reports about how your team is performing. What is the average sprint points per sprint and so on.
